# Medium Tags — 5 Sets

Pick ONE set. Medium allows exactly 5 tags per article.

---

## Set 1: The Builder's Architecture (targets engineers, CTOs)

1. Software Architecture
2. Web3
3. Progressive Web Apps
4. Hackathon
5. Full Stack Development

**Why:** Broadest reach. "Software Architecture" signals substance. "Progressive Web Apps" catches the PWA audience. "Hackathon" sets expectations appropriately.

---

## Set 2: The Web3 Developer Angle (targets crypto builders)

1. Web3
2. Blockchain
3. Hackathon
4. Open Source
5. Ethereum

**Why:** Positions squarely in Web3 territory. "Ethereum" is the highest-traffic blockchain tag. "Hackathon" frames the scope honestly — readers won't expect production-grade security.

---

## Set 3: The Indie Hacker / Solo Builder (targets founders)

1. Indie Hacking
2. Next.js
3. Web3
4. PWA
5. Startup

**Why:** Targets founders and solo builders who care about shipping speed and full-stack capability. "Next.js" + "PWA" shows modern stack knowledge. "Startup" catches the founder audience.

---

## Set 4: The AI-Augmented Build Story (targets tooling enthusiasts)

1. Artificial Intelligence
2. Software Architecture
3. Web3
4. Build In Public
5. Open Source

**Why:** Leads with AI to catch developers interested in AI-augmented workflows, then backs it with architecture credibility. "Build In Public" catches the indie/transparency crowd.

---

## Set 5: The Football + Tech Crossover (targets sports tech)

1. Sports Technology
2. Web3
3. Hackathon
4. PWA
5. Mobile App Development

**Why:** Niche targeting. Lower traffic but higher relevance — readers who find this via "Sports Technology" are already interested in the intersection. Good if you want to position for sports-adjacent gigs.

---

## My Recommendation

**Set 1** for maximum reach among technical readers who might hire you for full-stack Web3 work.

**Set 4** if you want to lead with the AI-augmented development angle (stronger differentiation, smaller audience).
